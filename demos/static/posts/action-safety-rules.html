Action Safety Rules for Owner-Approved Automation

Date: May 14, 2026

Tags: action, security, approval, operations

OpenVila actions are powerful because they can change website state. The operational model should always keep the owner in the approval loop.

Rule 1: actions are owner-created only

Users can request actions but cannot create or modify scripts. This prevents privilege escalation through prompt injection.

Rule 2: all risky actions require manual approval

Content blocking, refund execution, and account updates should always go through pending review and explicit owner confirmation.

Rule 3: keep an audit trail

Store action requests with timestamps, request payload, and final decision. This is useful for both debugging and compliance checks.

Rule 4: validate input in script layer

Even after approval, scripts should validate domain objects and constraints before writing data or updating content.
